# 低电-短见-畅梦人分级记忆 README

这组文件用于让新的 agent 快速学习三个连贯 replay 的模组构建方法。阅读顺序从高层到低层，不要跳过“转场类型”和“信息载体”，因为这两项决定线索是否能被玩家合理接住。

## 分级结构

1. `01_梗概与线索构建.md`
   - 最快入口。
   - 说明三部曲整体问题域、三篇各自的简化梗概、线索构建方法。
   - 用于回答“这三个模组在学什么、整体怎么搭”。

2. `02_内容呈现手法与语言风格.md`
   - 内容文档。
   - 专门整理网页、报告、录音、答辩录像、新闻、电台、闲聊、邮件、战术简报等呈现手法。
   - 用于学习“如何把线索写成真实资料，避免干瘪情报条”。

3. `03_三部曲模组纲要.md`
   - 三篇简纲。
   - 每一段只保留关键线索、转场理由和转场类型。
   - 用于快速查“玩家为什么会去下一个地点、这个跳转是否需要补强”。

4. 下级结构梳理
   - `../低电/低电_跑团replay结构梳理.md`
   - `../短见/短见_跑团replay结构梳理.md`
   - `../畅梦人/畅梦人_跑团replay结构梳理.md`
   - 这里是线性流水账，保留每一步调查员当时知道什么、因为什么去哪里、信息如何逐步串联。

5. 原 log/OCR
   - `../低电/抽取结果/低电_OCR校对版.md`
   - `../短见/抽取结果/短见_OCR校对版.md`
   - `../畅梦人/抽取结果/畅梦人_OCR校对版.md`
   - 当结构梳理仍不够细，回到这里查措辞、对话、检定、原始顺序。

## 转场类型

- `[实证转场]`：前一步已经给出充分线索或任务压力，调查员自然会去下一处。
- `[推想转场]`：线索存在，但需要调查员进行大胆联想、经验判断或主动押注。
- `[艺术插入]`：replay 为了节奏或美感突然给出资料、检索、旁线或文档。改写成可跑模组时，必须补上触发条件、检定入口、NPC 来源或失败代价。

## 使用规则

1. 写新模组时先读 `01_梗概与线索构建.md`，确定要模仿哪种线索结构。
2. 设计调查流程时读 `03_三部曲模组纲要.md`，检查每个转场是否能被玩家自然做出。
3. 写资料、网页、报告、录音、访谈时读 `02_内容呈现手法与语言风格.md`，优先模仿载体，不要只给结论。
4. 涉及具体情节点时回到三篇结构梳理。不要把后文真相提前写进前文。
5. 仍然不确定事实时回原 log。事实优先于漂亮概括。
